# Dev Log

## 2026-07-15

The `llm-backends` package was born and hardened in a single day. It started as an extraction from StoryDaemon (initial 0.0.1 drop), then went through the big "hardening merge" that absorbed the analyzer and NovelWriter backend variants into one coherent library (0.1.0), followed by two quick follow-ups to fix a fallback-model footgun and restructure the README (0.1.1). The through-line was consolidating three divergent copies of the same LLM-backend code into a single package with a unified model registry, provider hardening, and a real test suite (38 to 85 tests over the day).

**Decisions & notes:** The headline behavior change for adopters is that CLI-agent backends now strip their provider's API key(s) from a copy of the environment by default (`strip_provider_keys=True`), so agent CLIs authenticate via their own login instead of silently billing metered keys; the opt-out is per-interface. The merge unified 20 primary models plus an alias table covering StoryDaemon and NovelWriter spellings behind a public `resolve_model()`, made `ANTHROPIC_API_KEY` canonical while still honoring `CLAUDE_API_KEY` with a one-time DeprecationWarning, and added a Codex bubblewrap/user-namespace workaround for hardened Linux plus OpenRouter client hardening (retries, timeout, `openrouter:<upstream>` prefix passthrough). Several merges were deliberately *not* taken and recorded as such: the analyzer's app-level `*-cli` registry keys, its `max_tokens` default change, and NovelWriter's `claude-4-5-opus` alias (all deferred to later migration steps to avoid silently perturbing frozen defaults or upgrading models). NovelWriter's ignored-timeout bug was intentionally not copied. Consumers now read `llm_backends.DEFAULT_API_MODEL` instead of hardcoding `"gpt-5.5"` fallback literals in multiple files. The whole suite passes in an SDK-free venv, preserving the lazy-import contract.
